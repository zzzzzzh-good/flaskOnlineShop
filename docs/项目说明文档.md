# 说明文档

## 1. 项目简介

本项目是一个基于 **Flask** 框架构建的在线商城学习示例，涵盖用户认证、商品展示、评论发表等基础电商功能。项目采用 Flask Blueprint 进行模块化拆分，适合用于学习以下知识点：

- Flask 应用工厂模式与蓝图组织方式
- SQLAlchemy ORM 数据库操作
- Flask-Login 用户会话管理
- Werkzeug 密码哈希安全存储
- Jinja2 模板继承与上下文传递

---

## 2. 功能列表

| 功能 | 说明 |
|------|------|
| 用户注册 | 填写用户名和密码完成注册，密码以哈希形式存储 |
| 用户登录 | 凭用户名和密码登录，登录状态通过 Session 维持 |
| 用户注销 | 清除登录状态，跳转首页 |
| 商品列表 | 首页展示所有商品的卡片（图片、名称、价格） |
| 商品详情 | 展示商品完整信息及用户评论列表 |
| 发表评论 | 登录用户可在商品详情页提交文字评论 |

## 3. 项目文件说明

### `run.py` — 启动入口

```python
app = create_app()          # 调用工厂函数创建 app 实例
db.create_all()             # 首次运行时自动建表
create_fake_data()          # 若数据库为空，从 data/products.json 读取并插入商品
app.run(debug=True, ...)    # 启动开发服务器
```

`create_fake_data()` 的核心逻辑：

```python
with open('data/products.json', encoding='utf-8') as f:
    items = json.load(f)
products = [Product(**item) for item in items]  # 将每条 JSON 记录展开为关键字参数
db.session.add_all(products)
db.session.commit()
```

`Product(**item)` 是 Python 的字典展开语法，等价于逐一传入 `name=`、`price=`、`description=`、`image_url=` 四个字段。

### `data/products.json` — 商品数据

存放所有初始商品信息，格式为 JSON 数组，每个对象对应一件商品：

```json
[
  {
    "name": "商品名称",
    "price": 999.00,
    "description": "商品描述",
    "image_url": "https://图片地址"
  }
]
```

字段与 `Product` 模型一一对应，直接通过 `Product(**item)` 构建模型对象写入数据库。修改商品数据只需编辑此文件，无需改动任何 Python 代码。

### `app/__init__.py` — 应用工厂

`create_app()` 是整个应用的组装中心：
1. 创建 Flask 实例并设置配置
2. 将 `db`、`login_manager` 两个扩展绑定到 app（`init_app` 模式）
3. 导入并注册 `auth`、`shop` 两个蓝图

之所以在函数内部才 `import` 蓝图，是为了**避免循环导入**：蓝图文件需要导入 `db`，而 `db` 定义在 `__init__.py`，若在顶层导入蓝图会形成循环。

### `app/models.py` — 数据模型

定义三个数据库模型类，每个类对应一张数据表：

**`User`** — 用户

```python
class User(UserMixin, db.Model):
    id, username, password_hash
```

继承 `UserMixin` 提供 Flask-Login 所需的 `is_authenticated`、`get_id()` 等方法。

**`Product`** — 商品

```python
class Product(db.Model):
    id, name, price, description, image_url
    comments  # 反向关系，通过 product.comments 访问该商品所有评论
```

**`Comment`** — 评论

```python
class Comment(db.Model):
    id, content, date_posted
    user_id    # 外键，关联 user.id
    product_id # 外键，关联 product.id
    user       # 关系对象，通过 comment.user.username 取用户名
```

**`load_user` 函数**

```python
@login_manager.user_loader
def load_user(user_id):
    return db.session.get(User, int(user_id))
```

Flask-Login 每次请求时会调用此函数，根据 Session 中存储的 `user_id` 查询数据库，还原 `current_user` 对象。

### `app/routes/auth.py` — 认证蓝图

包含三个路由：

| 路由函数 | 逻辑说明 |
|----------|----------|
| `register()` | POST 时检查用户名是否已存在，存在则提示错误；否则哈希密码后写入数据库 |
| `login()` | POST 时查询用户名，再用 `check_password_hash` 比对密码，成功则调用 `login_user()` |
| `logout()` | 调用 `logout_user()` 清除 Session，需要 `@login_required` 装饰器保护 |

### `app/routes/shop.py` — 商城蓝图

包含两个路由：

| 路由函数 | 逻辑说明 |
|----------|----------|
| `index()` | 查询全部商品，传入模板渲染商品卡片列表 |
| `product_detail()` | GET 时渲染商品详情；POST 时判断是否登录，已登录则写入评论 |

### `app/templates/` — 模板文件

所有页面继承自 `base.html`，使用 Jinja2 的 `{% extends %}` 和 `{% block %}` 机制：

```
base.html（导航栏 + Flash 消息 + 页脚）
    ├── index.html      ← {% block content %} 商品列表
    ├── product.html    ← {% block content %} 商品详情 + 评论
    ├── login.html      ← {% block content %} 登录表单
    └── register.html   ← {% block content %} 注册表单
```

## 4. 关键概念解释

### 为什么使用 `init_app` 而不是直接传 `app`？

```python
# 方式一：直接绑定（不灵活）
db = SQLAlchemy(app)

# 方式二：init_app 模式（本项目使用）
db = SQLAlchemy()
db.init_app(app)
```

`init_app` 模式下，`db` 对象在模块顶层创建但不绑定任何 app，直到 `create_app()` 运行时才绑定。这样 `models.py` 可以安全地从 `__init__.py` 导入 `db`，而不触发循环导入。

### Flask-Login 的 `current_user` 是什么？

`current_user` 是 Flask-Login 提供的全局代理对象：
- 已登录时：代理的是 `load_user()` 返回的 `User` 实例
- 未登录时：代理的是匿名用户对象（`is_authenticated` 为 `False`）

模板中可直接使用：`{% if current_user.is_authenticated %}`

### Flash 消息机制

```python
# Python 中写入消息
flash('注册成功，请登录', 'success')  # 第二个参数是 Bootstrap 样式类

# base.html 中统一读取并展示
{% for category, message in messages %}
    <div class="alert alert-{{ category }}">{{ message }}</div>
{% endfor %}
```

Flash 消息存储在 Session 中，**只在下一次请求时显示一次**，之后自动清除。

## 5. 数据流示例：用户发表评论

```
用户填写评论表单 → 点击提交
        ↓
POST /product/1
        ↓
shop.product_detail() 处理请求
        ↓
检查 current_user.is_authenticated
    ├── False → flash 提示 + 重定向到登录页
    └── True  → 从 request.form 取出 content
                    ↓
                创建 Comment 对象（content, user_id, product_id）
                    ↓
                db.session.add() + db.session.commit()
                    ↓
                flash('评论发表成功') + 重定向回商品详情页
```
