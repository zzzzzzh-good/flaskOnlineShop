# 系统设计文档

## 1. 技术栈

| 层次 | 技术 | 版本 | 作用 |
|------|------|------|------|
| Web 框架 | Flask | 3.x | 路由、请求处理、模板渲染 |
| ORM | Flask-SQLAlchemy | 3.x | 数据库操作抽象层 |
| 认证 | Flask-Login | 0.6.x | 会话管理、登录状态维护 |
| 密码安全 | Werkzeug | 3.x | 密码哈希与校验 |
| 数据库 | SQLite | 3.x | 本地文件数据库 |

```

  │

  │
  ▼

  ├── 注册蓝图 auth
  └── 注册蓝图 shop

        │
```

## 4. 数据库设计

### 4.1 ER 图

```
┌──────────┐         ┌──────────┐         ┌──────────────┐
│   User   │         │ Comment  │         │   Product    │
├──────────┤         ├──────────┤         ├──────────────┤
│ id (PK)  │◄──┐ ┌──►│ id (PK)  │◄──┐     │ id (PK)      │
│ username │   │ │  │ content  │   └─────►│ name         │
│ password │   └─┤  │ date     │         │ price        │
│ _hash    │     │  │ user_id  │(FK)     │ description  │
└──────────┘     │  │ product_id│(FK)    │ image_url    │
                 └──┘           │        └──────────────┘
                                └──────────────►(一对多)
```

### 4.2 数据表说明

**user 表**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, 自增 | 用户唯一标识 |
| username | VARCHAR(100) | UNIQUE, NOT NULL | 用户名 |
| password_hash | VARCHAR(200) | NOT NULL | pbkdf2:sha256 哈希后的密码 |

**product 表**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, 自增 | 商品唯一标识 |
| name | VARCHAR(100) | NOT NULL | 商品名称 |
| price | FLOAT | NOT NULL | 商品价格（元） |
| description | TEXT | NOT NULL | 商品描述 |
| image_url | VARCHAR(200) | 默认占位图 | 商品图片地址 |

**comment 表**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, 自增 | 评论唯一标识 |
| content | TEXT | NOT NULL | 评论内容 |
| date_posted | DATETIME | 默认当前时间 | 发表时间（UTC） |
| user_id | INTEGER | FK → user.id | 评论作者 |
| product_id | INTEGER | FK → product.id | 所属商品 |

### 4.3 表关系

- `User` → `Comment`：一对多（一个用户可发表多条评论）
- `Product` → `Comment`：一对多（一个商品可有多条评论）

## 5. 路由设计

| 蓝图 | 路由 | 方法 | 是否需要登录 | 说明 |
|------|------|------|------------|------|
| shop | `/` | GET | 否 | 首页，展示所有商品 |
| shop | `/product/<id>` | GET | 否 | 商品详情页 |
| shop | `/product/<id>` | POST | 是（软拦截） | 提交评论 |
| auth | `/register` | GET | 否 | 注册页面 |
| auth | `/register` | POST | 否 | 处理注册表单 |
| auth | `/login` | GET | 否 | 登录页面 |
| auth | `/login` | POST | 否 | 处理登录表单 |
| auth | `/logout` | GET | 是（强制） | 退出登录 |

> **软拦截**：POST 评论时若未登录，重定向到登录页并提示，而非返回 403。
> **强制**：`/logout` 使用 `@login_required` 装饰器，未登录直接跳转到登录页。

## 6. 核心设计模式

### 6.1 应用工厂模式（Application Factory）

`create_app()` 函数负责创建并配置 Flask 实例，而非在模块顶层直接创建。

**好处**：
- 避免循环导入（`db`、`login_manager` 先创建，再通过 `init_app` 绑定到 app）
- 便于后续创建多个不同配置的 app 实例（如测试环境）

### 6.2 蓝图模式（Blueprint）

将路由按功能拆分为独立模块，最后注册到 app。

```
auth 蓝图  ──注册──►  app
shop 蓝图  ──注册──►  app
```

`url_for` 中必须加蓝图名前缀：`url_for('auth.login')`、`url_for('shop.index')`。

### 6.3 数据与代码分离（JSON 种子数据）

商品初始数据存放在 `data/products.json`，与代码完全分离：

```
data/products.json  ──读取──►  run.py create_fake_data()  ──写入──►  shop.db
```

`run.py` 启动时读取 JSON，动态构建 `Product` 对象批量插入数据库：

```python
with open('data/products.json', encoding='utf-8') as f:
    items = json.load(f)
products = [Product(**item) for item in items]
db.session.add_all(products)
```

**好处**：新增或修改商品只需编辑 JSON 文件，无需改动任何 Python 代码。

### 6.4 密码安全

用户密码不以明文存储，注册时使用 `generate_password_hash` 生成哈希，登录时使用 `check_password_hash` 比对：

```
注册：原始密码 → pbkdf2:sha256 哈希 → 存入数据库
登录：原始密码 + 数据库哈希 → check_password_hash → 布尔值
```
